👨‍🎓 Students
POST /students → register student (name, email, image → embedding)
GET /students → list students
GET /students/:id → get one
PUT /students/:id → update info
DELETE /students/:id → delete
🏫 Classrooms (camera mapping)
POST /classrooms → add room (camera_url)
GET /classrooms
GET /classrooms/:id
PUT /classrooms/:id
DELETE /classrooms/:id
📚 Subjects
POST /subjects
GET /subjects
GET /subjects/:id
PUT /subjects/:id
DELETE /subjects/:id
🕒 Sessions (core control)
POST /sessions/start
body: { subjectId, classroomId, startTime, endTime }
POST /sessions/end
body: { sessionId }
GET /sessions → list sessions
GET /sessions/:id → details
GET /sessions/active → current running session(s)
🎥 Recognition / Attendance (AI hook)
1) Recognize face (from camera/app)
POST /recognition
input: image (base64/file), sessionId
output: { studentId, confidence }
2) Mark attendance (called after recognition)
POST /attendance
{ studentId, sessionId, confidence }
3) Get attendance
GET /attendance/session/:sessionId
GET /attendance/student/:studentId
4) Summary (useful for UI)
GET /attendance/summary?sessionId=...
📩 Recheck Requests (your smart feature)
POST /recheck
{ studentId, sessionId, message }
GET /recheck → list (teacher/admin)
POST /recheck/:id/resolve
{ status: approved|rejected, note }

👉 (Agent can auto-resolve before teacher sees it)
